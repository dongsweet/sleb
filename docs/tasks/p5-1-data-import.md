# P5-1：数据迁移

| 字段 | 值 |
|------|------|
| 编号 | P5-1 |
| 优先级 | P5 — 数据迁移 |
| 预估工时 | 4h |

## 背景

原站有大量数据（建筑、技术、服务等目录条目，新闻、公告等）需要迁移到新平台。

## 目标

1. 从原站抓取/导出数据
2. 转换为新平台的格式
3. 导入到 PostgreSQL

## 涉及文件

1. **新建** `packages/db/src/migrations/import-data.ts` — 数据导入脚本
2. **新建** `apps/worker/src/imports.ts` — 批量导入处理器（部分）

## 数据来源

| 数据 | 来源 | 格式 |
|------|------|------|
| 建筑目录 | 原站 API 或爬虫 | JSON/CSV |
| 技术目录 | 原站 API 或爬虫 | JSON/CSV |
| 服务目录 | 原站 API 或爬虫 | JSON/CSV |
| 新闻/公告 | 原站内容 | HTML → Markdown |
| 出版物 | MinIO/S3 迁移 | PDF |
| 术语词典 | 静态数据 | JSON |

## 导入流程

```
1. 从原站获取数据 → 2. 数据清洗和转换 → 3. 批量导入数据库
4. 图片/文件上传到 MinIO → 5. 重建搜索索引
```

## 实现要点

- 分批导入，每批 100 条
- 事务操作保证原子性
- 数据校验：必填字段、唯一约束
- 导入报告：成功/跳过/错误统计
- 搜索索引在导入完成后批量重建

## 验收标准

- [ ] 建筑数据导入成功
- [ ] 技术/服务数据导入成功
- [ ] 新闻/公告数据导入成功
- [ ] 搜索索引重建
- [ ] 导入报告生成
