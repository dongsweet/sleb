# P4-5：邮件模板

| 字段 | 值 |
|------|------|
| 编号 | P4-5 |
| 优先级 | P4 — 通知 |
| 预估工时 | 3h |

## 背景

系统需要在各种场景下发送邮件通知。需要设计统一的邮件模板系统。

## 目标

实现邮件模板和发送逻辑，覆盖以下场景：

## 邮件场景

| 场景 | 触发条件 | 收件人 | 内容 |
|------|---------|--------|------|
| 注册验证 | 用户注册 | 新用户 | 验证链接 |
| 密码重置 | 忘记密码 | 用户 | 重置链接 |
| 订阅确认 | 订阅邮件列表 | 订阅者 | 确认链接 |
| 会员申请结果 | 管理员审批 | 申请者 | 通过/拒绝通知 |
| 支付成功 | Airpay 回调确认 | 会员 | 会员激活通知 |
| 联系表单 | 用户提交 | 管理员 | 新消息转发 |
| 会员到期提醒 | 到期前 30 天 | 会员 | 续费提醒 |
| 报告完成 | 报告生成完毕 | 会员 | 报告下载链接 |

## 涉及文件

1. **新建** `apps/worker/src/email.ts` — 邮件发送器
2. **新建** `apps/worker/src/templates/` — HTML 邮件模板
3. **新建** `apps/api/src/lib/notify.ts` — 通知工具函数

## 模板设计

每个模板是一个 HTML 文件，使用模板变量：

```html
<!-- templates/verify-email.html -->
<h1>Verify your email</h1>
<p>Click the link below to verify your email address:</p>
<a href="{{ VERIFY_LINK }}">Verify Email</a>
<p>This link expires in 24 hours.</p>
```

## 实现要点

- 使用 nodemailer 发送
- HTML 模板使用 `handlebars` 或简单字符串替换
- 响应式邮件模板（兼容各邮箱客户端）
- SLEB 品牌风格（颜色、Logo）
- 队列机制：推入 `email` 队列后 worker 处理
- 发送日志记录
