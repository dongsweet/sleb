# P2-2：BEEA 建筑能效评估

| 字段 | 值 |
|------|------|
| 编号 | P2-2 |
| 优先级 | P2 — 评估工具 |
| 预估工时 | 6h |

## 背景

BEEA（Building Energy Efficiency Assessment）面向非住宅建筑业主。评估建筑整体能效，需要更多参数。

## 目标

实现建筑级能效评估的多步骤表单和计算。

## 涉及文件

1. **新建** `apps/web/app/beea/page.tsx` — 评估入口页
2. **新建** `apps/web/app/components/assessments/BeeaForm.tsx` — 多步骤表单
3. **新建** `apps/api/src/modules/assessments/beea.ts` — 计算逻辑
4. **已有** `packages/db/src/schema.ts` — `assessmentRecords` 表（type: beea）

## 表单步骤

| 步骤 | 标题 | 内容 |
|------|------|------|
| 1 | 建筑信息 | 类型、面积、年份、楼层数 |
| 2 | HVAC 系统 | 空调类型、容量、能效比 |
| 3 | 照明系统 | 照明类型、功率密度、控制方式 |
| 4 | 电梯系统 | 电梯数量、类型、载重 |
| 5 | 其他系统 | 冷水机组、锅炉、备用电源 |
| 6 | 建筑围护结构 | 玻璃幕墙比例、保温层 |
| 7 | 结果 | 综合评分、分项能耗、改进建议 |

## 计算逻辑

- 基于各系统参数计算能耗
- 使用建筑类型基准值进行对比
- 输出 EUI（kWh/m²/year）
- 生成能效等级（A/B/C/D/E）

## 实现要点

- 比 THEEA 更复杂，表单字段更多
- 需要 Standard 及以上会员权限
- 支持导入已有建筑数据（从 My Building 选择）
- 结果页包含：EUI 柱状图、系统占比饼图、改进建议
