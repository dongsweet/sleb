# P5-3：表单验证

| 字段 | 值 |
|------|------|
| 编号 | P5-3 |
| 优先级 | P5 — 前端增强 |
| 预估工时 | 3h |

## 背景

所有表单需要统一的前端验证体验。

## 目标

1. 统一的表单验证库集成
2. 所有表单的校验规则
3. 友好的错误提示

## 涉及文件

1. **新建** `packages/shared/src/validators.ts` — 共享验证规则
2. **修改** 所有表单组件 — 使用 `react-hook-form` + `zod`

## 验证规则

| 字段 | 规则 |
|------|------|
| 邮箱 | 格式校验、唯一性 |
| 密码 | 8+ 字符、大小写+数字 |
| 姓名 | 2-50 字符、无特殊字符 |
| 手机号 | 新加坡格式 +65 |
| 邮编 | 6 位数字 |
| 数字 | 范围校验（>0）|
| 文件 | 类型限制、大小限制 |

## 实现要点

- 使用 `react-hook-form` + `@hookform/resolvers/zod`
- 前端即时验证（onChange）
- 提交时完整校验
- 错误消息显示在对应字段下方

## 验收标准

- [ ] 所有表单有验证规则
- [ ] 错误提示清晰
- [ ] 即时验证生效
- [ ] 提交拦截正常
