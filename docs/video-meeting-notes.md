## 1. 数据库迁移

- **核心决策**：数据库可以完全复制一份一模一样的，甚至还可以优化
- 有完整服务器权限，可以拿到全部数据
- 原站使用 ASP.NET，源码可能不可见，但可以通过后台功能完全扒下所有功能
- 迁移策略：**增量同步** — 做一个系统一直运行着，有一点数据就同步一点，测试完成后一键切换域名

## 2. 数据导入

- 建筑数据：支持 Excel (.xlsx) 和 CSV 导入
- 数据无特殊字符，CSV 即可
- 当前电费账单数据仍需要 Excel 导入，不支持自动识别

## 3. AI 功能（重点新增）

### 3.1 AI Chatbot（首页核心功能）
- **首页改为 AI 对话框为主** — 类似 HuggingChat 的体验
- 用户可以直接问 AI 分析问题、获取推荐
- 背景显示 SLEB 楼宇图像，但主交互是 AI 对话

### 3.2 AI 知识库
- 为 AI 建立知识库，新产品上线时知识库自动更新
- 给 AI 设置"员工手册"（prompt），可以随时修改
- 政府推广某项技术或有补贴时，更新 prompt 让 AI 着重推广
- 会员上传新技术/新产品时，AI 自动分析 brochure 提取信息

### 3.3 AI 文档解析
- 上传的报告可能是多种类型，需要 AI 识别
- 未来目标：自动识别电费账单数据（当前仍需要 Excel 导入）

### 3.4 AI 推荐替代 Smart Advisor
- 原站 Smart Advisor 是写死的建议列表，修改困难
- **改为 AI 推荐** — 基于建筑数据生成个性化建议
- AI 可以根据知识库随时变化推荐内容

## 4. 评估工具算法

- 当前算法直接写死在程序里
- **需求**：需要在后台可以更换算法/调整参数
- 参数是固定的，但算法逻辑需要可配置
- 复杂度排序：THEEA（最简单）< SMEEA（中等）< BEEA（最复杂）

## 5. 楼宇数据接入

- 部分楼宇已有数据接入，通过 API 或 Excel 方式
- 数据源包括：楼宇管理系统、新加坡 SP（水电数据）
- 新系统需要保证 API 接口兼容，已有的数据接入方式不变
- 实际方式：一些楼宇用 Python 脚本跑 Excel，然后调用 API 提交

## 6. 每日数据同步

- 需要每天有一个 cron job 拉取最新数据
- 数据是实时更新的（每天都更新）

## 7. 会员系统

- 会员支付使用**日历支付**（非 Stripe）
- Airpay 费率：1.5%（之前 Stripe 是 3.4% + 5毛）
- 三级会员：Basic/Standard/Premium

## 8. 绿色金融

- OCBC 银行是主要合作方，天天都在用
- 绿色建筑可以获得绿色贷款
- 未来可能扩展到更多银行（DBS、UOB 等）

## 9. 性能问题

- 原站图片加载慢，ASP.NET 本身就比较卡
- 新系统使用 Next.js Image 优化 + CDN 解决

## 10. 项目管理

- 企业是主要客户
- 有调查问卷功能（类似 Google Map 上的调查）
- R&D 项目列表和地图需要保留

## 11. 多语言

- 原站支持中英文切换
- 新系统后续加上

## 12. 部署策略

- 先做测试网站，与原站并行运行
- 测试完成后，切换域名即可完成上线
- 不需要停机迁移，域名切换在一夜之间完成
